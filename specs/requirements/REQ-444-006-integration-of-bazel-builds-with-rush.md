# Integration of Bazel builds with Rush CLI commands

## Description
Integrate Bazel build support into the Rush CLI commands (`rush dev`, `rush build`, `rush deploy`) so that Bazel-based components are built and deployed alongside other component types.

## Specification

### Modified Files
1. `rush-cli/src/commands/build.rs` - Add Bazel build type handling
2. `rush-cli/src/commands/dev.rs` - Support Bazel components in dev mode
3. `rush-cli/src/commands/deploy.rs` - Deploy Bazel-built images

### Build Command Integration
In `build_component()` function:

```rust
async fn build_component(context: &BuildContext, toolchain: &Arc<ToolchainContext>) -> Result<()> {
    match &context.build_type {
        // ... existing build types ...
        
        BuildType::Bazel {
            location,
            output_dir,
            targets,
            additional_args,
            ..
        } => {
            debug!("Building Bazel component at {location}");
            
            // Use BazelStrategy for build execution
            let strategy = BazelStrategy::new();
            strategy.build(spec, context).await?;
            
            // OCI image is generated by the strategy
            // Load into Docker for local use
            load_oci_image_to_docker(output_dir, context).await?;
        }
        
        // ... rest of match arms ...
    }
    Ok(())
}
```

### Dev Command Integration
- File watching for Bazel source files
- Automatic rebuild when source files change
- Hot reload of containers with new images

### Deploy Command Integration
- Push Bazel-generated OCI images to registry
- Apply Kubernetes manifests for Bazel components

## Acceptance Criteria
- [ ] `rush build` successfully builds Bazel components
- [ ] `rush dev` runs Bazel components with file watching
- [ ] `rush deploy` pushes Bazel images and deploys to Kubernetes
- [ ] Bazel build errors are displayed clearly to users
- [ ] Build caching works across invocations
- [ ] Component dependencies are respected (builds in correct order)
